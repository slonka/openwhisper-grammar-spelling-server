# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build and run

```bash
# First-time setup: download ONNX model + generate tokenizers
mise run setup-models

# Build
cargo build --release

# Run (default port 8787)
cargo run --release -- --port 8787

# Or use mise for the full chain (setup-models -> build -> run)
mise run run
```

CLI flags: `--port`, `--model-path`, `--tokenizer-path`, `--lt-url`
Env vars: `PORT`, `LT_URL`, `RUST_LOG`

No test suite exists. Use `cargo check` for quick compilation checks and `cargo clippy` for linting.

## Architecture

Rust rewrite of a Python grammar/spelling server. Exposes an OpenAI-compatible chat completions API (`POST /v1/chat/completions`) using Axum + Tokio.

Two models available:
- `text-cleanup-pipeline` - cleanup without translation (default)
- `text-cleanup-translate-pl-en` - cleanup + Polish-to-English translation

### Pipeline stages (sequential, in order)

All orchestrated in `src/pipeline.rs`. Each stage is in `src/stages/`:

1. **detect.rs** - Language detection via Lingua (English or Polish, defaults to Polish)
2. **fillers.rs** - Regex-based filler word removal (Polish + English verbal fillers)
3. **itn.rs** - Inverse text normalization (spoken numbers to digits). English uses text2num, Polish has custom implementation
4. **punctuation.rs** - ONNX model inference for punctuation restoration and capitalization (pcs_47lang.onnx)
5. **correction.rs** - Hardcoded regex rules for common spelling mistakes (per-language)
6. **config.rs** - User replacements hot-reloaded from `~/.config/openwhisper-cleanup/replacements.json` via file watcher
7. **grammar.rs** - LanguageTool API client, applies corrections end-to-start to preserve offsets
8. **translation.rs** - Optional Polish-to-English via Helsinki-NLP/opus-mt-pl-en (Candle framework, lazy-loaded from HuggingFace)

### Key patterns

- Pipeline stages degrade gracefully - if a stage fails, processing continues
- Translation model loads on first request, not at startup
- Config hot-reload uses `notify` file watcher with Arc+Mutex sharing
- HTTP handlers in `src/handlers.rs`, server setup in `src/main.rs`

## Required model files

These must exist before running (generated by `mise run setup-models`):
- `models/pcs_47lang.onnx` - punctuation/capitalization model (~233 MB)
- `models/tokenizer.json` - multilingual BERT tokenizer
- `models/tokenizer_pl_en.json` - Polish-English translation tokenizer

## External dependency

LanguageTool must be running (default: `http://localhost:8010/v2/check`). Grammar stage silently skips if unavailable.
